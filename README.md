# NTSB FOIA MU5735

This repository contains files originally shared by another GitHub user related to FOIA requests on the MU5735 investigation.

The original repository has since been deleted or made private by its owner. To protect their privacy, the files have been re-uploaded here without preserving the original commit history.

This repository serves as an archive and allow others contribute to maintain access to those materials.

An unofficial Chinese translation of the NTSB recorder report is available at:
https://github.com/adminlby/NTSB_FOIA_MU5735/blob/main/MU5735_NTSB_Recorder_Report_CN/MU5735_NTSB_Recorder_Report_Chinese.md

## NTSB FOIA public release

NTSB has published a related FOIA Reading Room release:

- Title: `DCA22WA102 3/21/2022 Wuzhou, China`
- Published date: 2026-05-01
- Format: ZIP
- Pages: 13
- Size: 326.80 MB
- Comments: `China Eastern Airlines Flight 5735 FOIA Release Records`. NTSB notes that it does not have any Cockpit Voice Recorder transcripts. The `.upk` files require specialized Flight Data Recorder analysis software, and even with that software the information would not be usable without the manufacturer's proprietary documentation.

Links:

- NTSB FOIA Reading Room: https://securefoia.ntsb.gov/app/ReadingRoom.aspx
- Direct attachment: https://securefoia.ntsb.gov/app/AddAttachment.aspx?docid=66&ispaldoc=F
- Internet Archive mirror: https://web.archive.org/web/20260502141354/https://securefoia.ntsb.gov/app/AddAttachment.aspx?docid=66&ispaldoc=F
- Source discussion: https://github.com/wrongly-cuddly-obsession/NTSB_FOIA_MU5735/issues/14

## Third-party projects

The following community projects are unofficial simulations or visual references based on public materials. They do not represent findings by NTSB, CAAC, or this repository.

- MU5735 Simulation Replay: https://mu5735.colorcard.cc  
  Repository: https://github.com/colorcard/MU5735-Simulation-Replay  
  This project describes a web replay of the final two minutes built from public ADS-B records and FDR export data released through FOIA. Its timeline uses the final ADS-B Reveal record as the anchor and looks back 120 seconds. The first roughly 26.8 seconds overlap with limited FDR data; after that, the replay enters an ADS-B-only segment, so FDR-only values such as attitude, controls, and engine parameters are visible only during the overlap.
- MU5735 3D flight reconstruction demo: https://djzoom.github.io/MU5735/  
  Repository: https://github.com/djzoom/MU5735

Source discussion: https://github.com/wrongly-cuddly-obsession/NTSB_FOIA_MU5735/issues/13

## 第三方项目

以下项目为社区成员基于公开资料制作的非官方模拟或三维重建，仅作直观参考，不代表 NTSB、CAAC 或本仓库的调查结论。

- MU5735 Simulation Replay：https://mu5735.colorcard.cc  
  项目仓库：https://github.com/colorcard/MU5735-Simulation-Replay  
  该项目基于 MU5735 公开 ADS-B 记录与 FOIA 公布的 FDR 导出数据，构建 ADS-B + FDR 融合数据到 Web 端的最后 2 分钟模拟回放。其说明称，主时间轴以 ADS-B Reveal 末条记录为基准向前回溯 120 秒，其中窗口前约 26.8 秒与 FDR 有限重叠；后段进入纯 ADS-B 区间，因此姿态、操纵、发动机等 FDR 专属量只在重叠段内可见，后段仅保留 ADS-B 可支撑的外部运动信息和严格派生量。
- MU5735 三维飞行重建 Demo：https://djzoom.github.io/MU5735/  
  项目仓库：https://github.com/djzoom/MU5735

来源：https://github.com/wrongly-cuddly-obsession/NTSB_FOIA_MU5735/issues/13
